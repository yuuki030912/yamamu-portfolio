# やまむーゲーム攻略Wiki

## プロジェクト概要
GitHub Pagesで運営するマルチゲーム攻略サイト。サブディレクトリ方式で複数ゲーム対応。
ドメイン: yamamu-youtube.jp
対応ゲーム: イナイレV (`/inazuma/`), ポケポケ (`/pokepoke/` 準備中)

## イナイレV用語集（正確な定義）

### ビルド（チームビルド / ビルドパッシブ / チームパッシブ）
各選手が持つ「チームパッシブ」のこと。選手ごとにビルドタイプが決まっており、チームメンバーの多数決でチーム全体の「チームビルド」が決定する。チームビルドによりチーム全体にバフ効果が適用される。

- 選手がスピリットから選手化する時にランダムでビルドタイプが決まる（キャラごとに出やすいタイプあり）
- レベル47でチームパッシブが全解放される
- レベル50でカスタムパッシブスロットが解放され、ビルドの幅が広がる

#### 7種類のビルドタイプ
1. **正義ビルド** — ファウルを取られるまでチームのATDF上昇（監督マネ統一で最大60%）。常時発動で安定性が高い。現環境で最強のビルド。初心者はまずこれ。ファウルするとバフ消滅が弱点
2. **テンションビルド** — フォーカス/スクランブル勝利時の獲得テンション上昇（上限150%）。テンション50%以上時フォーカスATDF上昇、テンション100%時シュートAT上昇。周回・CPU戦向き
3. **必殺ビルド** — シュートの必殺率が上昇し、必殺技のテンション消費を下げる効果。ステータス上昇効果がなく対人には不向き。格上CPU向け
4. **カウンタービルド** — キャッチ以外でボール奪還後15秒間チームのシュートAT上昇。ロングシュートとの相性が良い
5. **キズナビルド** — パスでキズナパワーを貯めやすくなる。ボール支配率が上がる
6. **ラフプレービルド** — ダッシュで吹き飛ばした時テンション上昇（上限+50%）＋ダッシュでのファウル率減少（上限-80%）。手動操作で強いが、AI監督モードでは活かせない玄人向け
7. **フリーダムビルド** — チーム内でビルドが偏らない場合のデフォルト。特定のビルド効果なし

#### ビルドパッシブの優先順位（やまむー推奨）
- **最強**: 正義（安牌。つけれるなら正義一択）
- **妥協時の次点**: テンション50%以上時の効果が正義に次いで有用
- **カウンター/テンション**: マネージャーにつけた方が効率が良い
- **必殺/キズナ/ラフプレイ**: 正義とはかけ離れすぎているので非推奨

#### ビルドの組み合わせ（Ver1.5.0以降）
Ver1.5.0でチームパッシブに上限が設けられ、単一ビルド特化よりも複数ビルドの組み合わせが主流に。
- **正義×テンション** — 正義で基礎ステを上げてフォーカスで勝ちやすくし、テンションで獲得TP量を増やす。安定と火力のいいとこ取り
- **正義×ラフプレー** — 正義のバフ維持しつつ、ファウル率減少でダッシュも使える
- **正義×カウンター** — ボール奪取後のシュートATアップでロングシュートが強力

※「正義テンション」= 正義ビルドが主流の環境、または正義×テンションのハイブリッド構成を指す通称。「技巧テンション」「ラフテンション」というゲーム内の正式な概念は存在しない。

### テンション（ゲーム内リソース）
必殺技を使うために必要なゲージ（エネルギー）。テンションを貯めるにはフォーカス勝負に勝つ必要がある。
- フォーカス勝利でテンション獲得（通常+60、テンションビルドで最大+150）
- フォーカス敗北でも少量（+30）獲得
- 必殺技を使うとテンションを消費する（強力な技ほど消費が多い）
- テンション30%以上を維持することが重要（特定能力の発動条件）
- テンション50%以上、100%時にはビルドによって追加効果あり

### その他の重要用語
- **パッシブ継承** — 他キャラのパッシブスキルを自キャラに付与するシステム。継承対象は選手のみ（監督・マネージャーは選手化で対応）。同キャラ・同ビルド制限なし、1キャラから1個だけ、継承元は消滅する。バサラ・ヒーロー・他チーム編成中キャラは素材不可

#### ポジション別おすすめパッシブ（パッシブ継承）
- **FW**: 敵陣での自身のシュートAT +8%（最優先）、同ポジションシュートAT +3.5%。フォーカス特化なら異属性フォーカスATDFや同属性フォーカスATDF
- **MF・DF**: 基本的に異属性フォーカスATDF、同属性フォーカスATDF
- **DF城壁**: 異属性フォーカスATDF、同属性フォーカスATDF
- **GK**: 異属性フォーカスATDF、同属性フォーカスATDF
- **フォーカス** — 攻撃側（AT）と守備側（DF）の数値で勝敗が決まるバトルシステム
- **BASARA化** — キャラクターを最上位レアリティに強化するシステム。入手方法は2つ：①マーケット「荒ぶる者たちの巣窟」でHEROスピリット8枚と交換（対象HEROのLv50以上が必要）、②バサラシード使用（Ver3.0.1で1個配布、どのレアリティからでもBASARA化可能）。チーム編成は1人のみ（理想はBASARA1+HERO2+レジェンダリー）。ビルドタイプの自由変更・「新たな可能性」による技の付け替えが可能
- **バサラシード** — HEROスピリット8枚分を肩代わりしてBASARA化できるアイテム。Ver3.0.1で1個配布。2個目以降の入手方法は現時点で不明（公式「かなり頑張る必要あり」）。一度使うと消費されやり直し不可。同キャラ2人目のBASARA交換はバサラシードのみ可能
- **キズナリンク** — 選手間の絆を深め、連携必殺技を発動するシステム
- **タクティクス** — 試合中の戦術設定（例：サイドラインスピア）
- **KP** — ゴールキーパーの重要ステータス（キーパーポイント）
- **熱血オーバードライブ** — フォーカス値を大幅に上げる特殊能力
- **チームパッシブ** — 各選手が持つビルドタイプに紐づくパッシブ。チーム全体に効果が適用される
- **カスタムパッシブ** — レベル50で解放される追加パッシブスロット

## サイト構造・技術仕様

### ファイル構成
```
/                        — ルート（共有アセット + ポータル）
├── index.html           — ゲーム選択ポータル
├── style.css            — 全ページ共通CSS（CSS変数でダーク/ライト対応）
├── video.css            — 動画個別ページ・カテゴリページ用CSS
├── theme.js             — テーマ切替JS
├── icon.png             — サイトアイコン
├── sitemap.xml          — サイトマップ（generate.htmlで生成）
├── robots.txt           — クロール設定
├── CNAME                — ドメイン設定
├── category-*.html      — リダイレクトスタブ（→ inazuma/）
├── guide-*.html         — リダイレクトスタブ（→ inazuma/）
├── video-inazuma-*.html — リダイレクトスタブ（→ inazuma/）
│
├── inazuma/             — イナイレV攻略
│   ├── index.html       — イナイレVトップ（Wiki風2カラム）
│   ├── script.js        — index.html用JS
│   ├── videos.js        — 動画データ配列
│   ├── videos.json      — 動画データ元ファイル
│   ├── generate.html    — 管理ツール（GAME_ID="inazuma", ASSET_PREFIX="../"）
│   ├── category-*.html  — カテゴリ・テーマ別ページ
│   ├── guide-*.html     — 攻略ガイドページ（手動作成）
│   └── video-inazuma-*.html — 各動画の個別ページ
│
├── pokepoke/            — ポケポケ攻略（準備中）
│   ├── index.html       — ポケポケトップ（準備中表示）
│   ├── script.js        — index.html用JS
│   ├── videos.js / videos.json — 動画データ（空）
│   └── generate.html    — 管理ツール（GAME_ID="pokepoke"）
```

### レイアウト構造（Wiki風・Game8/GameWith参考）
- **ポータルページ**: ゲーム選択カード（`.portal-grid` > `.portal-card`）
- **ゲームページ共通**: ヘッダー + `.game-nav`（ゲーム切替）+ `.wiki-nav`（ゲーム固有ナビ）
- **ヘッダー**: Row1（ロゴ + 検索 + テーマ切替）
- **コンパクトヒーロー**: `.wiki-hero`（タイトル + サブタイトル）
- **2カラム**: `.wiki-layout` > `.wiki-main`（メイン）+ `.wiki-sidebar`（サイドバー、1024px以下で非表示）
- **メインエリア**: ピックアップ → 攻略ガイドカード → テーマ別動画セクション(6テーマ×4件) → ショート動画セクション → SEOテキスト
- **サイドバー**: カテゴリナビ / 人気記事TOP5 / ガイド一覧 / SNSリンク
- **フッター**: `.footer-nav` で全ページリンク

### ナビバー（2層構造）
```html
<!-- ゲーム切替ナビ（全ゲーム共通） -->
<nav class="game-nav" aria-label="ゲーム選択">
  <a href="../index.html" class="game-nav-link">トップ</a>
  <a href="../inazuma/" class="game-nav-link active">イナイレV</a>
  <a href="../pokepoke/" class="game-nav-link">ポケポケ</a>
</nav>

<!-- ゲーム固有ナビ（ゲームごとに内容が異なる） -->
<nav class="wiki-nav" aria-label="メインナビゲーション">
  <a href="index.html" class="wiki-nav-link">ホーム</a>
  <a href="category-build.html" class="wiki-nav-link">ビルド攻略</a>
  ...
</nav>
```
※ 現在のページに対応するリンクに `class="wiki-nav-link active"` を付与
※ ゲーム固有ナビのリンクはサブディレクトリ内の相対パス

### テーマ分類ロジック（getThemeCategory）
動画のtagsを元に6テーマに自動分類。script.js / generate.html の両方に同じロジックが存在。
- `build`: パッシブ, ビルド
- `equipment`: 装備, ビーンズ
- `farming`: 周回, hero, 効率
- `update`: アップデート, basara, アプデ
- `beginner`: 初心者, フォーカス, キズナ
- `battle`: 対戦, テンション, タクティクス, 戦い方（デフォルト）

### CSS設計
- CSS変数で全色管理（`--bg-primary`, `--text-primary`, `--accent` 等）
- ダークモード: `[data-theme="dark"]` セレクタで変数上書き
- アクセントカラー: `--accent: #667eea`（紫青系）
- レスポンシブ: 1024px（サイドバー非表示）、768px（2列）、480px（1列）

### generate.html の使い方
1. videos.json を選択して読み込む
2. 「すべて生成」でZIPダウンロード（index.html, category-*.html, category-{theme}.html, video-*.html, sitemap.xml, videos.js）
3. 「新規のみ生成」で既存ファイルを除外して差分生成
4. guidePages配列にガイドページを手動登録（generate.html内、自動生成対象外）

### ページ追加時の手順
- **動画ページ**: `inazuma/videos.json` に追加 → `inazuma/generate.html` で再生成
- **ガイドページ**: `inazuma/` にHTMLを手動作成（パスは `../style.css` 等）→ generate.htmlのguidePages配列に追加 → index.htmlのガイドカード/サイドバー/フッターにリンク追加
- **新テーマ追加**: getThemeCategory()のタグ判定を追加 → themeConfig/categoryConfigに定義追加
- **新ゲーム追加**: ディレクトリ作成 → generate.htmlをコピーしてGAME_ID変更 → ポータルindex.htmlにカード追加 → game-navにリンク追加（全generate.htmlのgameNavHtml更新）
